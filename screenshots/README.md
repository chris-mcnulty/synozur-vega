## Screenshot Guidelines

### Technical Specifications
- **Format:** PNG (preferred) or JPG
- **Resolution:** Minimum 1920x1080 recommended
- **Aspect Ratio:** Maintain original aspect ratio
- **File Size:** Optimize for web (compress to ~500KB or less per image)

### Visual Guidelines
- Use a clean, production-ready environment (no test data with placeholder text)
- Include realistic sample data that demonstrates the feature
- Ensure proper lighting/theme (light mode preferred for consistency)
- Highlight or annotate key UI elements when helpful (use subtle annotations)
- Crop to show relevant UI section while maintaining context
- Blur or anonymize any sensitive company information

### Content Guidelines
- Use professional, realistic example data
- Company values examples: "Innovation", "Integrity", "Customer Focus"
- Objective examples: "Become market leader in our segment"
- Key Result examples: "Increase market share from 15% to 25%"
- Avoid Lorem Ipsum or obviously fake data

## How to Capture Screenshots

1. Set up a clean demo environment with realistic sample data
2. Navigate to the appropriate page/feature
3. Ensure the UI is in the desired state (e.g., forms filled in, data loaded)
4. Use screenshot tool (macOS: Cmd+Shift+4, Windows: Windows+Shift+S, or browser extension)
5. Save with the exact filename specified above
6. Optimize/compress the image if needed
7. Verify the screenshot displays correctly in the USER_GUIDE.md
